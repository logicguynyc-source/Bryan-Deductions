Bryan Deduction

Bryan-Style Proof Editor
Bryan Deduction is a Fitch-style natural deduction system extended with a truth predicate. It uses a quotational syntax with no restrictions on formula content. Key differences from standard systems:
  • From any accessible formula φ, you can derive True(⌜φ⌝). And from any accessible True(⌜φ⌝), you can derive φ.
  • From any accessible line n, you can derive True(⌜n⌝). And from any accessible line that states True(⌜n⌝), you can derive the formula on line n.
  • Subderivations can explore what follows from assuming True(⌜n⌝) or ¬True(⌜n⌝) for proof lines n — including self-referential or forward-referential ones — as long as the assumption is falsifiable.
The key innovation: assumptions must be falsifiable — meaning there must exist a coherent assignment of truth values under which the assumption is false. For ordinary formulas this just means the formula isn't a tautology. For self-referential formulas like True(⌜n⌝) on line n, it means the assignment of truth values to the relevant lines must be internally consistent (a line can't be deemed both true and false) and must make the assumption false. This constraint is designed to prevent liar sentences, Curry sentences, and similar self-referential tricks from being used to derive a contradiction from the inference rules alone.

How to Use

1. Set up — Enter comma-separated premises and a conclusion, then click Create Proof.

2. Type formulas — Each row has a formula cell (left) and a justification cell (right). Click either to edit.

3. Justifications — Type the rule abbreviation and line numbers, e.g. ∧I 1,2 or →E 3,4. Premises are labelled automatically.

4. Subproofs — Use +sub to open a subproof (needed for →I, ¬I, ∨E, ∀I, ∃E). The first line is always an assumption.

5. FOL quantifiers — Write ∀x Fx or ∃x Gxy. For ∀E and ∃I specify the substitution: ∀E 1 [x/a] (replace x with a). For ∀I write ∀I 3-5 [c/x] (c is the arbitrary constant, x the variable). For ∃E write ∃E 1 2-4. Scope convention: binary connectives (→, ∨, ∧, ↔) always bind more tightly than quantifier scope extension. So ∃x Dx→Q means (∃x Dx)→Q. Write ∃x(Dx→Q) to bind → inside the quantifier.

6. Check — Click Check Proof to validate. Each line turns green (valid) or red (error).

TFL shortcuts — - or ~ for ¬, /\ for ∧, \/ for ∨, -> for →, <-> for ↔, _|_ for ⊥.

FOL shortcuts — See the FOL Syntax panel for formula syntax and substitution notation.

Truth predicate — Write True(⌜φ⌝) (formula) or True(⌜n⌝) (line number) to assert truth. TI m: from φ on line m, derive True(⌜φ⌝) or True(⌜m⌝). TE m: from True(⌜φ⌝) or True(⌜n⌝) on line m, derive the named formula. Click ⌜ ⌝ in the symbol bar to insert corner quotes.

FOL Syntax Guide
Recursive definition of a WFF
Base cases — each of the following is a WFF:
  1. ⊥ — the falsum constant
  2. Any single uppercase letter (A–Z) standing alone — a propositional variable
  3. Any single lowercase letter (a–z) standing alone — a sentential variable
  4. An uppercase letter followed by one or more lowercase term letters (e.g. Fx, Gxy) — a predicate application. (Ax and Ex are reserved for quantifier shorthands and are not WFFs.)
  5. t1 = t2 where t1 and t2 are lowercase term letters — an identity statement
  6. True(⌜n⌝) where n is a positive integer — the truth predicate applied to a line-number reference. (The proof-time constraint that line n must contain a WFF is a semantic condition, not a grammatical one.)
Recursive cases — if φ and ψ are WFFs and x is a variable, then so are:
  1. True(⌜φ⌝) — the truth predicate applied to a quasi-quoted WFF
  2. ¬φ
  3. (φ ∧ ψ), (φ ∨ ψ), (φ → ψ), (φ ↔ ψ)
  4. ∀x φ, ∃x φ
Nothing else is a WFF.

Quasi-quotation & Truth
⌜φ⌝Name of formula φ (a term)
True(⌜φ⌝)Truth predicate: φ is true
True(⌜n⌝)Truth predicate: the formula on proof line n is true (n must refer to the line of a WFF)
∀x True(⌜Fx⌝)Every Fx-instance is true
⌜φ⌝ is a quasi-quotation term denoting φ. Unlike opaque quotation, variables inside ⌜⌝ still participate in substitution: instantiating x→a in ∀x True(⌜Fx⌝) yields True(⌜Fa⌝). Use the ⌜ ⌝ buttons in the symbol bar above.
